Tokenization in NLP

Explore and compare three fundamental approaches to breaking text into tokens — the building blocks of every language model.

What is Tokenization?

Before a language model can process text, the raw string must be split into discrete units called tokens. The strategy used to create these tokens profoundly affects vocabulary size, sequence length, and the model's ability to handle unseen words (OOV — Out-Of-Vocabulary).

This page explores three popular tokenization strategies, using a German literary corpus (Selbstbetrachtungen by Marcus Aurelius) as a running example.

The Three Approaches

Word

Word-Level Tokenization

The simplest strategy: split on whitespace and treat every unique word form as a separate token.

  • ✓ Semantically meaningful tokens
  • ✓ Short sequences
  • ✗ Huge vocabulary
  • ✗ Cannot handle unseen words (high OOV)

Vocab size on corpus: ≈ 3 649

Character

Character-Level Tokenization

Each character in the text becomes its own token. The vocabulary is tiny (the alphabet + punctuation) but sequences become very long.

  • ✓ Tiny vocabulary
  • ✓ Zero OOV — any text can be encoded
  • ✗ Very long sequences
  • ✗ Tokens lack semantic meaning

Vocab size on corpus: ≈ 83

BPE

Subword (BPE) Tokenization

Byte Pair Encoding iteratively merges the most frequent adjacent character pairs. The result is a balanced vocabulary of subword units.

  • ✓ Balanced vocabulary size (configurable)
  • ✓ Zero OOV
  • ✓ Moderate sequence length
  • ✗ More complex training step

Vocab size on corpus: ≈ 1 000 (configurable via num_merges)

How BPE Works — Step by Step

BPE starts from a character-level vocabulary and builds up subword units by repeatedly merging the most frequent pair:

  1. Split every word in the corpus into individual characters, appending a special end-of-word marker </w>.
  2. Count the frequency of every adjacent character pair across the vocabulary.
  3. Merge the most frequent pair into a single new token.
  4. Repeat steps 2–3 for num_merges iterations (e.g., 1 000 times).

Key Takeaways

  • Word tokenization yields short, meaningful sequences but struggles with unknown words and creates enormous vocabularies.
  • Character tokenization eliminates the OOV problem entirely but produces very long sequences, making it harder for models to learn long-range patterns.
  • BPE (subword) tokenization strikes the best balance: a tuneable vocabulary size, no OOV tokens, and moderate sequence lengths. This is why virtually all modern large language models (GPT, LLaMA, etc.) use subword methods.
